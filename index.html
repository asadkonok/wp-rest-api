Getting Started

Rest API will provide JSON data based on passing parameters.

API END POINT: 

 Posts: http://example.com/wp-json/wp/v2/posts/

 Categories: http://example.com/wp-json/wp/v2/categories/

 Taxonomies: http://example.com/wp-json/wp/v2/taxonomies/

 Tags: http://example.com/wp-json/wp/v2/tags/

Get Posts

API END POINT TO GET Posts: 

http://example.com/wp-json/wp/v2/posts/

API needs to have right parameters to get right data. Here is the list of available parameters.

Parameter Description Example Default Required
context Scope under which the request is made; determines fields present in response. view or embed or edit view Yes
page Specify the page of results to return. It returns the second page of posts results. 1 or 2 or 3 or 4 or etc None Yes
per_page Specify the number of records to return in one request, specified as an integer from 1 to 100. It will return only the first post in the collection. 1 or 10 or etc 10 Yes
search Limit results to those matching a string. For searching in posts "some text" None No
after Limit response to posts published after a given ISO8601 compliant date none None No
author Limit result set to posts assigned to specific authors. Author id can be used. 2 None No
author_exclude Ensure result set excludes posts assigned to specific authors. Author id can be used. 2 None No
before Limit response to posts published before a given ISO8601 compliant date. none None No
exclude Ensure result set excludes specific post IDs. 27 None No
include Limit result set to specific post IDs. 27 None No
offset Offset will use the default number of posts per page 6 or 12 etc None Yes
order Control whether results are returned in ascending or descending order. asc or desc desc Yes
orderby Control the field by which the collection is sorted date or relevance or id or include or title or slug date Yes
slug Limit result set to posts with one or more specific slugs. Use comma when more slug will be used. cras-ac-purus-vitae-neque-lacinia None No
status Limit result set to posts assigned one or more statuses. publish,pending,draft publish No
categories Limit result set to all items that have the specified term assigned in the categories taxonomy. Multiple categories can be separated by comma. 2 or 2,3 None No
categories_exclude Limit result set to all items except those that have the specified term assigned in the categories taxonomy. Multiple categories can be separated by comma. 2 or 2,3 None No
tags Limit result set to all items that have the specified term assigned in the tags taxonomy. Multiple tags can be separated by comma. 6,9 None No
tags_exclude Limit result set to all items except those that have the specified term assigned in the tags taxonomy. 6 None No
sticky Limit result set to items that are sticky. true or false None Yes

Get Categories

API END POINT TO GET Categories: 

http://example.com/wp-json/wp/v2/categories/

API needs to have right parameters to get right data. Here is the list of available parameters.

Parameter Description Example Default Required
context Scope under which the request is made; determines fields present in response. view, embed, edit view Yes
page Current page of the collection. 1 or 2 or etc 1 Yes
per_page Maximum number of items to be returned in result set. 10 or 12 10 Yes
search Limit results to those matching a string. some text None No
exclude Ensure result set excludes specific IDs. Multiple IDs can be separated by comma. 2 or 2,3 None No
include Limit result set to specific IDs. Multiple IDs can be separated by comma. 2 or 2,3 None No
order Order sort attribute ascending or descending. asc or desc asc Yes
orderby Sort collection by term attribute. id or include or name or slug or term_group or description or count name Yes
hide_empty Whether to hide terms not assigned to any posts. true or false None Yes
parent Limit result set to terms assigned to a specific parent. Parent terms return its child terms. Parent terms id will be used. 2 or 3 or etc None Yes
post Limit result set to terms assigned to a specific post. 27 or 28 or etc None Yes
slug Limit result set to terms with one or more specific slugs. Multiple slugs can be separated by comma. featured or uncategorized or featured,uncategorized None No

Get Taxonomies

API END POINT TO GET Taxonomies:

Parameter Description Example Default Required

Get Tags

API END POINT TO GET Tags: 

http://example.com/wp-json/wp/v2/tags/

API needs to have right parameters to get right data. Here is the list of available parameters.

Parameter Description Example Default Required
context Scope under which the request is made; determines fields present in response. view, embed, edit view Yes
page Current page of the collection. 1 or 2 or etc 1 Yes
per_page Maximum number of items to be returned in result set. 10 or 12 or etc 10 Yes
search Limit results to those matching a string. some text None No
exclude Ensure result set excludes specific IDs. Multiple IDs can be separated by comma. 6 or 9 or or 6,9 or etc None Yes
include Limit result set to specific IDs. Multiple IDs can be separated by comma. 6 or 9 or or 6,9 or etc None Yes
offset Offset the result set by a specific number of items. 6 or 12 or etc None Yes
order Order sort attribute ascending or descending. asc or desc asc Yes
orderby Sort collection by term attribute. id or include or name or slug or term_group or description or count name Yes
hide_empty Whether to hide terms not assigned to any posts. true or false None Yes
post Limit result set to terms assigned to a specific post. None None Yes
slug Limit result set to terms with one or more specific slugs. Multiple slugs can be separated by comma test1 or test1,test2 None Yes

Example Call

Here is the simplest form of API call

Posts:

http://example.com/wp-json/wp/v2/posts/?categories=2&per_page=2&order=asc&orderby=date

Categories:

http://example.com/wp-json/wp/v2/categories/?exclude=1&per_page=8&order=asc&orderby=name

Taxonomies:

http://example.com/wp-json/wp/v2/taxonomies/?type=post

Tags:

http://example.com/wp-json/wp/v2/tags/?exclude=6&hide_empty=true&per_page=8&order=asc&orderby=name
